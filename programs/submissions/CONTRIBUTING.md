# Contributing to the Package Manager Repository

Thank you for your interest in contributing to this package manager! To keep the project secure and maintain quality, please follow the guidelines below.

## How to Contribute

1. **Fork the repository**  
    Create your own copy of the repository by clicking the "Fork" button on GitHub.

2. **Create a new branch**  
    Make a branch for your contribution:
    ```bash
    git checkout-b your-feature-name
    ```

3. **Add your program**  
- Place your Python program in the `programs/` folder.  
- Each program must be contained in its own folder inside `programs/`.  
- Include a README file describing the program and any usage instructions.  
- Ensure your program does not perform malicious actions.

4. **Test your program**  
Make sure your program works with the package manager and does not break anything.

5. **Create a pull request**  
Push your branch to your fork and create a pull request (PR) to this repository.

## Review Process

- All submissions will be reviewed by the maintainers or moderators.  
- Only after approval will the program be merged into the main repository and become available for installation.  
- We reserve the right to reject any submission that is malicious, poorly documented, or does not meet the quality standards.

## Security Guidelines

- Do not include any code that executes arbitrary shell commands.  
- Programs must be safe and not harm users or their systems.

## Code of Conduct

- Be respectful and constructive in your communication.  
- Follow GitHub community guidelines.

Thank you for helping improve this project!
